## Slide 2

Role-based Open Access Model  -- User Journey

Data Consumer  = An individual that accesses and analyses approved R&D data within governance & privacy controls

Key Message

"Much more data, much faster, open by design"

Key Outcomes

More Data:  Greater (& growing!) pool of data proactively available, removing barriers

More Responsibility:  Unlocking data puts greater responsibility & trust on data users

Contact Email:  Users will initially be contacted via email confirming inclusion in the  Role-based Open Access Model  rollout

SharePoint Page:  Users will be able to access general comms materials from the  Transforming Data Sharing & Access  SharePoint page ( HERE )

User Guide:  The contact email will provide users with a link to the User Guide -- detailing the full ole-based Open Access Model -- and the mandatory training required to access data

Provisioning:  Upon completing mandatory training -- R&D Data Office will perform access provisioning steps

Additional guidance can be found through comms & training materials in SharePoint (Transforming Data Sharing & Access -- page)

If individual assistance is required contact the DPO team within the R&D Data Office

ROAM Dashboard:  View available studies & users included in scope through Power BI dashboard 

Keep an eye out for new innovative tools being rolled out for ROAM!

Self-Review:  Before initiating a new project, check intended use aligns with guidance in User Guide -- if assistance is required contact R&D Data Office as in support section

Access Mechanisms:  Follow guidance on access mechanisms (Data Product Solution (PDP) for accessing data

Project Completion Form:  Once a project is completed, it is the responsibility of the Data Consumer / Lead to complete a short form stating results
