# 2025 Baseline - Workstream Synthesis

---

## Overview

The 2025 baseline documents establish the foundation for ROAM (Role-based Open Access Model), including the core guidance, user guides, agreement templates, and regulatory SOPs that govern all data sharing under the 90:10 model.

## ROAM Model Summary

**Definition:** Role-based Open Access Model - access paradigm for scale and speed across R&D

**Core Concept:** Top-down approval of 'data collections' and 'terms of use' - groups of users granted open access to collections of data

**Key Difference:** Users continuously covered by top-down agreement, removing need for repetitive individual approvals

## Key Terms

| Term | Definition |
|------|------------|
| **OAC** | Open Access Collection - specified scope of data that consumers can potentially access |
| **AOT** | Data Use Terms - formal document defining terms of use for an OAC |

| **DCM** | Data Collection Manager (role in Collectoid) |

### One-Time Setup:
1. **Awareness** - Contact email, Knowledge Hub SharePoint
2. **Compliance** - Complete 4 mandatory trainings, access provisioned

1. **Ideation** - Use ROAM Dashboard to find available data
2. **Pre-planning** - Self-check against User Guide (Primary Use)
3. **Execution** - Access via PDP
4. **Close-down** - Submit Project Completion Form

## Permitted Uses (Primary Use)

All ROAM collections permit:
- Understand how study drug works in body
- Better understand studied disease
- Develop diagnostic tests
- Learn from past studies to plan new studies
- Improve scientific analysis methods
- Drug development, regulatory compliance
- Safety monitoring
- Cross-study learning
- Scientific publications (with PSO)
- AI analytics

---

## Prohibited Uses

All ROAM collections prohibit:
- AI development/training
- Software development/testing
- External data sharing (without standard process)
- Commercial use
- Attempting to identify individuals
- Combining with external datasets without approval

---

## Data Provisioning Checks

Before access granted, DPO verifies:
1. **Drug Product Rights** - M&A, divestments, collaborations
2. **Cross Border Data Transfer** - HGR, US Data Security Rule
3. **Data Owner Approval** - Studies >6mo post-CDL open by default

## Agreement Structure

### OAC Agreement defines:
- Collection ID and name
- Data scope (TAs, study status, data types)
- Inclusion/exclusion criteria
- Opt-in/opt-out process

### AOT Agreement defines:
- Base definition (ID, name, description)
- Reference to OAC
- Terms and conditions of use
- User scope (by department and role)

---

## Roles & Responsibilities

| Role | Responsibility |
|------|----------------|
| **Data Owner/DDO** | Approve OAC and AOT; accountable for data |
| **Data Consumer Lead** | Accountable for staff's work within AOT bounds |
| **R&D Data Office Lead** | Own creation of OAC/AOT; facilitate setup |
| **Data Consumer** | Complete training; use data within terms |

---

## Relevance to Collectoid

The 2025 baseline defines WHAT Collectoid must manage:

1. **Collections** - OAC Agreements with scope definitions
2. **Terms** - AOT Agreements with user/use restrictions
3. **Users** - Role-based access per AOT
4. **Compliance** - Training requirements, permitted uses
5. **Opt-in/Opt-out** - Quarterly review process
