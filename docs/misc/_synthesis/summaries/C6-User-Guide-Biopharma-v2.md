## Access Mechanisms (BioPharma-specific) `[ref: Section 3.3]`

### 1. Data Product Solution (PDP)
Same as generic ROAM:
- SPDP pipeline with SDTM standard data in Starburst
- Environments: SCP, Domino, AI Bench, entimICE
- Query on demand, no bulk copy

### 2. Entimice Access Process (BioPharma-specific)
- **Purpose:** Access original study assets in native platforms
- **Process:**
  1. Collections Manager - DO ensures eligible studies/users listed per DDO approval
  2. Grant Folder Access - Submit request to create user management group mapped to study folders
  3. Check Data Inclusion - Eligible users verify what data they have access to
  4. Continuous Use - Access remains open until study/user removed
  5. Continuous Monitoring - Collection Managers monitor changes
  6. Access Permissions - User groups updated to reflect membership/study changes

---

## Access Environments `[ref: Section 3.2.3]`

- Uses existing R&D environments users already have access to (Domino, Entimice, etc.)
- NOT introducing new analysis environments as part of this model
- All data usage must remain within approved analytics environment boundaries

---

## Permitted Uses `[ref: Section 3.2.1]`

Full IMI wording list (same as other ROAM guides):
- Conduct trials, regulatory compliance
- Drug development, safety monitoring
- Cross-study learning, scientific analysis
- Publications (following PSO process)
- AI analytics (excluding AI development/training)

---

## Prohibited Uses `[ref: Section 3.2.2]`

- AI development/training
- External data sharing
