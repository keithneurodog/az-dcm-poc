## Entities/Concepts Defined

### Catapult PoC `[ref: slides 11-14]`
- **Definition:** "A conversational agent that answers questions on the new open data collections" `[ref: slide 11]`
- **Key question it answers:** "What can I access now, why/why not, and how do I proceed?"
- **Focus:** Access and Data Discovery (other themes identified for future)

**Catapult Stakeholder Feedback:**
- ODSAI priority: Search and access clarity
- Bridge Scientific Intelligence + 90/10 + policy
- Sci Int Scale constraint
- Scoped approach

**Catapult Lessons Learned:**
- 90/10 was still evolving - moving target for automation/agentification
- Policy interpretation amplifies the issue
- Source selection and readiness challenges
- Answer irregularities
- Evaluation depends on defined journeys

### Keith's Collectoid Proposal `[ref: slide 16]`

**Key insight: "Agentic ≠ Chat"**

Problems with AI chat interfaces:
- Natural language is ambiguous and difficult for some users
- AI can be slow and unresponsive
- AI makes mistakes and can be biased

**Approach: "AI-assisted, not AI-replaced"**
- AI suggests → User confirms
- Intent becomes UI, not a black box
- Existing UI patterns stay intact

### Sherlock Definition `[ref: slide 31]`
The go-to place for any users to:
- Get an overview of all the R&D data available in AZ (DATA)
- Get a better understanding of the R&D open data landscape (DATA)
- Find references to the sources with more detailed information
- Find information about the status of R&D DO services requests (SERVICES)
- Get an overview of all the R&D DO services & processes (SERVICES)

### Scope Components `[ref: slides 22-28]`
Four dimensions:
1. **Scope:** What is included?
2. **Working group:** How is it delivered?
3. **Approach:** How does the solution look like?
4. **Design/Implementation:** How is it executed?

Visual shows two main tools: **Collectoid** and **Sherlock**
