## Primary Use Reinterpretation `[ref: slides 3-6]`

### Key Changes (June 2025 DGB Decision)
- Retiring Extended Primary Use (EPU)
- Full Primary Use now governs data use within Drug Development
- Expands available patient data
- Removes time-consuming sub-setting
- Delivers broader, faster access within consent & privacy safeguards

### Primary Use Scope
- **Across all data types:** Clinical, omics, imaging, operational, safety, diagnostic, digital health
- **Beyond Therapeutic Area:** Multi-TA disease, organ-based, comorbidities, adverse events, digital characteristics

### AI Clarification
- **AI analytics = Primary Use** (permitted)
- **AI development/training = Secondary Use** (not permitted under ROAM)

## ROAM Overview `[ref: slides 7-13]`

### 90:10 Principle
- **90%:** User needs open by default for qualified use
- **10%:** Request-based for complex edge cases

### Key Components
| Component | Description |
|-----------|-------------|
| Primary Use Reinterpretation | Leverages June 2025 DGB decision |
| Open Access Collections | Studies/data grouped into pre-approved collections |
| Data Use Terms | Approves user groups, role-aligned intent, permitted use |
| Access Solutions | R&D approved platforms (Domino) + new mechanisms (Immuta) |
| Collection Management | Joint stewardship by Data Office and TAs |

## Access Mechanisms `[ref: slide 14]`

### Data Product Solution (PDP)
- **Overview:** Scalable access via pre-prepared Data Products (standardized tables)
- **Access Method:** Proactive (pre-provisioned after training)
- **Data Focus:** PDP - Enriched Clinical Primary Data Product (SDTM via Starburst)
- **Environments:** Entimice (SAS), SCP, Domino, AI Bench

### Entimice Access Process
- **Overview:** Flexible route for on-demand use of original study assets
- **Access Method:** Reactive (granted on demand)
- **Data Focus:** Original SAS files, omics data, imaging data
- **Note:** "Discussions ongoing regarding open access to LSAF (updates in Q1 2026)"

## Decision Tree for ROAM `[ref: slide 16]`

**Step 1: Included User?**
- Is Data Consumer included in AOT scope & ROAM rollout phase?
- If No → Use IDAP or IEMT

**Step 2: Use Case Assessment**
- Does intent align with IMI "supporting AZ drug development" categories?
- (Lists 12 permitted use cases from AI analytics down to publications)

**Step 3: Exceptions Check**
- Does use case involve:
  - External data sharing?
  - AI development?
  - System/application testing?
  - New/unconsidered technologies?
- If Yes to any → Not permitted under ROAM

**Step 4: Proceed**
- If all checks pass → Access via ROAM
