## Data Plane {#slide-2}

2

The R&D Data Plane serves as an integrated data storage and information management platform supporting research and development activities.

 The platform is structured around four foundational pillars: 

Data Exchange    

Data Storage

Data Access  

Self-service 

Manages  both internal and external data movement, ensuring secure, access-controlled, and compliant bi-directional data transfers to/from  DataStore . ​

Technical enablement of access to data within the workspaces through PBAC (Policy-based Access Control) and ABAC (Attribute-based Access Control). ​

Manages on-demand data storage, known as Workspaces, which enables users to automatically request compliant storage solutions for different source systems and applications. ​

Self-Service Portal operates as an interface for users to interact with the  DataPlane  platform. It provides both a User Interface (UI), and an API, catering for both human and programmatic interaction.

## Data Provisioning Process {#slide-13}

13

Requester needs to provide the below details to get access for the ingested data:

- What data modalities are needed to be provisioned? 
- Who will be analysing this data (who needs access to this data)? 
- How long will this data access be needed for?  
- What would be the analysis intent and methodology? 
- Are the results from this analysis going to be used for any Internal Decision Making, Publication/External Communication?  
- Will AI/ML be used? If yes, will it be to build or train a model? 
- Where/which analysis platform should this data be provisioned to?
