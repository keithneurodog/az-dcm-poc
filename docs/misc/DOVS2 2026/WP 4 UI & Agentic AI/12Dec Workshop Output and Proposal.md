## 2026 strategies supported by AI UI/UX {#slide-16}

Challenge Diagnosis

Solution Guiding policy

Proposal Actions

Users and provisioning teams struggle to explore and manage data access collections, preventing proactive data access

- Provisioning teams create, manage and verify data collections manually
- There is no tool to assist users to view and manage data collections

Provide end-to-end visibility and control of data collections, enabling proactive access management through automation and continuous monitoring

- Develop an interface to explore, manage, and monitor data collections, access rules, and user groups
- Bring together capabilities to manage open-access collections and data access requests
- Use agent-based automation to support provisioning teams to monitor and propose collection changes
- Leverage AI and chatbots to support the enquiry of data collections and the explanation of data access decisions

Data collections

Users have difficulty finding and exploring the data they need, preventing data reuse

- The information we manage is complex and overwhelming, as are the platforms and services used to find, explore, and access data
- Information from different platforms and services is not interlinked
- There is no single point where all the information can be found

Provide a conversational reasoning system integrating content from different systems to answer user questions that enable users to find and explore data

- Develop a chatbot interface to allow user to enquiry, contextualize and summarize information available in different systems
- Integrate the content from exploration platforms, discovery platforms, user support portals and the data collections platform with the chatbot
- Use  agentification  to synthesize and act upon high-context information across disparate sources.

Virtual assistant

Less than 5% of R&D data is catalogued centrally making difficult for users to find data

- Multiple uncoordinated cataloguing initiatives with semi-manual pipelines
- Cataloguing delegated to teams without incentives, knowledge or accountability
- No clear view of all R&D data resources available in AZ
- Focus on deep metadata for few assets instead of broad coverage

Establish a single, accountable capability to coordinate the cataloguing of R&D data, prioritizing coverage over metadata depth and scaling cataloguing activities with automation

- Create funded central cataloguing team focused on metadata collection, harmonization, validation, remediation and integration following minimum information standards
- Development, refactoring and maintenance of scalable cataloguing pipelines with the same tech stack
- Identify and register all R&D data sources across platforms and domains
- Use agentification to automate the cataloguing steps that require manual effort
- Help the remediation team visualize the status of the pipelines and where are the main issues of quality that require remediation

Users struggle to understand which access tools, processes, services, and information are available, which hinders productivity

- Information about services, tools, processes and requests to find and access data ​are scattered and can be overwhelming
- Documentation available is not always up to date or presented in a user-friendly manner the user understands
- 

Provide a conversational reasoning system that integrates documentation from different systems to answer user questions about services, tools, processes & requests 

- Develop and chatbot interface to allow user to enquiry, contextualize and summarize documentation available in different systems
- Integrate documentation from different access platforms and services
- Work with access platforms and services to improve documentation to improve the efficiency of the chatbot
- Use  agentification  to understand, synthesize, and act upon high-context documentation across disparate sources.
